[실행(메인) 파일 안내]

1) 자바스크립트 실행(엔트리 포인트)
- 메인 실행 파일: index.js
- npm start를 실행하면 index.js가 실행됩니다.

2) index.js와 server.js 관계
- index.js는 서버 실행을 위해 src/server.js의 main()을 호출합니다.
- src/server.js는 서버를 실제로 띄우는 파일이며, 단독으로도 실행 가능합니다.

[실행 방법]

A) npm으로 실행(권장)
- npm start

B) node로 직접 실행
- node index.js

C) 포트 변경(기본 3000)
- node index.js --port 3001

[관련 파일]
- index.js: 실행 엔트리
- src/server.js: 서버 시작(main), 포트 파싱
- src/app.js: Express 앱 구성/라우트 연결

하드코딩은 비추천하여 아직 진행하지않았음.


[주의 사항 (의도적 취약점 데모)]

- 이 프로젝트에는 “스캔 데모를 위한 의도적 취약점 코드”가 포함되어 있습니다.
- 특히 src/vuln_samples/ 아래 파일들은 “스캔 전용(fixture)”이며 앱 실행 경로(라우팅/require)에 연결하지 않습니다.
- 실서비스/과제 제출용 안전 코드로 사용하면 안 됩니다. 스캐너(Semgrep/Gitleaks/로컬 스캔)에 탐지되는 것이 정상입니다.
- 하드코딩 시크릿은 실제 키가 아닌 데모용 문자열이지만, 저장소에 올리면 정책/스캐너에 의해 탐지될 수 있습니다.


[취약점 스캔 방법]

1) 로컬 데모 스캐너(이 레포 제공)
- 스캔 대상: src/ 이하의 .js 파일 전체(= src/vuln_samples 포함)
- 실행(권장): node scripts/scan.js
	- PowerShell에서 npm 실행 정책 문제 발생 시에도 위 명령은 동작합니다.
- 기대 결과: Findings: 20 (사진의 6개 항목 포함 + 추가 항목)

2) Semgrep (로컬 룰 사용)
- 룰 파일: .semgrep.yml
- 예시 실행: semgrep --config .semgrep.yml

3) Gitleaks
- 예시 실행: gitleaks detect --source .
- (PII/데모 패턴까지 더 강하게 탐지하려면) gitleaks detect --config .gitleaks.toml --source .


[서버 실행 방법(참고)]

- 서버 실행은 취약점 스캔과 별개입니다.
- 실행: npm start (또는 node index.js)
